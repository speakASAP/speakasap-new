# Marathon frontend — Legacy UI refactor plan

**Date:** 2026-02-22  
**Last updated:** 2026-02-22  
**Status:** Phases A–G implemented (full plan execution complete).  
**Goal:** Make the marathon frontend (marathon.alfares.cz) look exactly like the legacy marathon UI in speakasap-portal, so the new SPA is visually and structurally aligned with the legacy design.

**Reference:** Legacy marathon lives in [speakasap-portal](https://github.com/speakASAP/speakasap-portal/tree/stable): `marathon/templates/marathon/`, `marathon/static/`, and portal `static/` (Bootstrap, fonts, img/bg).

**Constraints:** No changes to database-server, auth-microservice, nginx-microservice, logging microservice. All configuration only in marathon and flipflop codebases; prod nginx is regenerated by deploy scripts.

---

## Current state

- **Marathon frontend** (`marathon/frontend/`): Vite + React + TypeScript; all routes implemented (Home, Landing, Winners, Profile, Step, etc.).
- **Problem:** Frontend has minimal styling: generic `index.css` (system font, basic container) and `landing.css` with a **dark slate theme** that does **not** match legacy. Home is a plain list; internal pages (Winners, About, Reviews, etc.) are unstyled lists/buttons. No shared header/footer, no legacy typography, no legacy colors or layout.
- **Legacy:** Bootstrap 3 + custom `marathon.css` (ProximaNova/GothamPro, green CTAs, language hero backgrounds, blue stripe sections, circle blocks, review carousel, tabs, white sections, footer with schema.org).

## Step-by-step plan

Execute in order. Mark items with ✅ when done and update this doc.

### Phase A: Design system and global shell

- [x] **A.1** **Design tokens**  
  Create a single source for legacy-aligned tokens (e.g. `frontend/src/styles/tokens.css` or variables in `index.css`):
  - Colors: body `#ecf0f1`; primary green gradient `#29a55e` → `#6fc400`; stripe `#14b5ea`; nav accent `#50d0de`; text on dark `#fff`; muted `#757575`.
  - Font stacks: ProximaNova (with fallback to system sans); GothamPro for brand if available.
  - Spacing/section padding to match legacy (e.g. promo min-height 523px, section padding).
- [x] **A.2** **Typography and base**  
  - Add ProximaNova/GothamPro (self-host from marathon `public/fonts/` or use fallback; legacy uses `../fonts/ProximaNova-*.otf`, `GothaPro*.otf`).
  - Set `body`: background `#ecf0f1`, font-family ProximaNova/sans-serif, 16px.
  - Ensure headings use same family (legacy: ProximaNova for h1–h5).
- [x] **A.3** **Assets**  
  - Copy or link legacy assets into `marathon/frontend/public/` (or `marathon/public/` so NestJS serves them):
    - Hero backgrounds: `img/bg/de.jpg`, `en.jpg`, `cz.jpg`, `es.jpg`, `tr.jpg`, `it.jpg`, `fr.jpg`, `pt.jpg`, `nl.jpg`, `pl.jpg`, `no.jpg` (from speakasap-portal `static/img/bg/` or `marathon/static/img/bg/`).
    - Landing images: `img/landing/` (step-1…step-5, grammar/materials/talk/result pngs, review/quote/chevron/hand/purse/point/phone/mail, etc.) from `speakasap-portal/marathon/static/img/landing/` if present, or portal `static/`.
  - Document in plan or README: if assets stay in portal repo, either copy once to marathon or serve via CDN/URL (config in marathon only).
- [x] **A.4** **Global layout shell**  
  - Add a shared layout component (e.g. `Layout.tsx`): same header and footer for all pages.
  - **Header:** Fixed navbar matching legacy: logo "SpeakASAP®" (link to `/`), nav links (Финалисты, Отзывы, О марафоне, Правила, Помощь, Мой профиль, Регистрация, Награды, Поддержка), green "Начать" (to `/register` or first language landing). Mobile: hamburger, collapse menu. Use semantic `<header>` + `<nav>`; classes aligned with legacy (e.g. `.navbar`, `.navbar-fixed-top`, `.affix` behavior via scroll).
  - **Footer:** Three columns: company name + address/contacts (can be simplified or full schema.org); social icons (YouTube, VK, Facebook, Telegram, Instagram); link to policy. Copyright line. Reuse structure from `block11_footer.html`; links to marathon.alfares.cz or speakasap.com as per product.
- [x] **A.5** **Wrap App with Layout**  
  - In `App.tsx`, wrap `<Routes>` with `<Layout>` so every route gets the same header/footer. Optionally allow a "bare" layout for landing (no extra nav) if design requires; otherwise one shell for all.

**Done when:** All pages share the same header and footer; body and typography match legacy; tokens and assets are in place.

### Phase C: Landing page (`/:langSlug/`)

Landing already has correct structure (promo, results, program, reviews, video, certificates, FAQ, guarantee, motivation, registration, footer). Refactor **styling** to match legacy, not behavior.

- [x] **C.1** **Promo block**  
  - Full-width hero with **language-specific background image** (e.g. `url(/img/bg/de.jpg)` for `langSlug=de`). Dark overlay; min-height ~523px; centered title and subtext (white, text-shadow). Green CTA "Начать бесплатно*" and small print "*Через N дня…".
  - Nav inside promo: same as global nav but with smooth-scroll to #steps, #reviews, #tour, #faq, #about (or #register). Class `promo lang-{code}` for bg.
- [x] **C.2** **Stripe "Что вы получите"**  
  - Background `#14b5ea`; heading white. Four circle blocks (Основы грамматики, Эксклюзивные материалы, Разговорная практика, Результат) with legacy icons (grammar, materials, talk, result). Desktop: expand-on-hover circles; mobile: stacked. Use legacy classes/copy from `block2_results.html` and `marathon.css` (.stripe, .nav-main, .circle-wrapper, .circle.grammar/.materials/.talk/.result).
- [x] **C.3** **Program block**  
  - "Программа марафона" with **tabs**: Определение знаний и целей, Базовая грамматика и лексика, Упражнения, Аудирование, Общение с носителем. Each tab: image (step-1…step-5) + text. Copy from `block3_program.html`. Optional: price block + green CTA below.
- [x] **C.4** **Reviews block**  
  - "Отзывы наших студентов". Legacy: photo strip + one review with triangle. New: keep API `GET /api/v1/reviews`; style as legacy (white section, .review, .photos, .triangle). If multiple reviews, carousel or "next" button.
- [x] **C.5** **Video, certificates, FAQ, guarantee, motivation**  
  - Video: same as now (iframe); section title and spacing like legacy.  
  - Certificates: short block, legacy copy.  
  - FAQ: accordion with plus/minus icons; copy from `block7_faq.html`.  
  - Guarantee: legacy block9_guarantee copy and style.  
  - Motivation: block8_motivation copy; CTA button.
- [x] **C.6** **Registration form**  
  - Keep existing `RegistrationForm` and API; style form to match legacy (e.g. .form-registration padding, labels, green submit button).
- [x] **C.7** **Landing footer**  
  - Same as global footer (company, socials, policy, copyright). Can reuse Layout footer or inline for landing.

**Done when:** Landing at `/de/`, `/en/`, etc. looks like legacy: same promo bg, stripe, tabs, reviews, FAQ, footer.

### Phase F: CSS strategy and cleanup

- [x] **F.1** **Single design system**  
  - Prefer one global stylesheet (or few) that implement legacy: e.g. merge/refactor `index.css` + `landing.css` into legacy-aligned theme (light bg, green CTAs, blue stripe, white sections). Remove or override dark slate theme from landing.
- [x] **F.2** **Bootstrap vs custom**  
  - Option A: Add Bootstrap 3 (or 5 with compatibility) to marathon frontend for grid and navbar/tabs/modal if it speeds parity. Option B: Keep no Bootstrap; implement grid and components with CSS Grid/Flexbox and legacy class names. Document choice.
- [x] **F.3** **Icons**  
  - Font Awesome 4 (or 5) for social/footer/UI icons to match legacy; or inline SVG. Material Icons only if used on legacy (e.g. winners).
- [x] **F.4** **Responsive**  
  - Legacy uses 768px breakpoints; ensure navbar collapse, promo text size, circle layout, and tabs work on mobile like legacy.

**Done when:** No duplicate or conflicting themes; one coherent legacy look; responsive.
