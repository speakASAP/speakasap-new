## 2. Detailed Issues and How to Solve

### 2.1 Notifications microservice — 405 Not Allowed / timeouts (CRITICAL for email)

**Logs:** `speakasap-portal/logs/ses.log`

**Pattern:**

- `405 Client Error: Not Allowed for url: https://notifications.statex.cz/notifications/send`
- `HTTPSConnectionPool(host='notifications.statex.cz', port=443): Read timed out. (read timeout=10)`

**Recent:** Jan 28–31; affects "Ticket created" and "Отправлено уведомление о неоплаченном счете".

**Cause:** Portal calls notifications microservice; server returns 405 (method not allowed) or timeout.

**Fix:**

1. Confirm notifications microservice expects **POST** to `/notifications/send` and that nginx/proxy allows it (no GET-only rule).
2. In **speakasap-portal** (or config that defines the client): ensure request method and URL match the notifications API (only in allegro/speakasap codebase, not prod nginx if regenerated by deploy).
3. If timeout: check notifications service health and response time; do **not** blindly increase timeout (per user rules); fix slow processing or add retries with backoff.
4. Verify on prod: `curl -X POST https://notifications.statex.cz/notifications/send ...` (with valid body/headers).

### 2.4 Helpdesk ticket delete — "operator does not exist: integer = uuid"

**Logs:** `app.log`  
**Pattern:** `Django delete() failed for ticket XXXXX due to cascade issue, using raw SQL: operator does not exist: integer = uuid`

**Cause:** A FK in the helpdesk model is integer but references a UUID column (or vice versa); Django cascade delete builds a query that mixes types.

**Fix:**

1. In **speakasap-portal/helpdesk**: find the view that deletes tickets and the model(s) involved (ticket, follow-up, etc.).
2. Check migrations: ensure FK types match (integer↔integer or uuid↔uuid). If a column was changed to UUID, add a migration to fix the type and/or the raw SQL fallback.
3. If raw SQL is used as fallback, ensure it uses correct type (e.g. `::uuid` cast in PostgreSQL where needed).

### 2.6 CSSUTILS — invalid background / vendor prefixes

**Logs:** `app_errors.log`, `app.log`  
**Pattern:** `Invalid value for "CSS Level 2.1" property: url() repeat #BBDEFB` and "Unknown Property name" for `-ms-text-size-adjust`, `-webkit-*`, etc.

**Cause:** Inline CSS in email (or premailer) uses `background: url() repeat #BBDEFB` (invalid shorthand) and vendor prefixes that cssutils doesn’t recognize.

**Fix:**

1. Fix the email template CSS: use valid `background` syntax (e.g. `background-color` + `background-image` separately; avoid invalid shorthand).
2. Or: when using cssutils only for inlining, suppress or redirect its logger to DEBUG so these don’t appear as ERROR in app_errors.
